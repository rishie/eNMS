The ansible playbook service executes the specified playbook on a single device at a time (using ansible-playbook '-i' argument. The options parameter allows to user to pass key/value variable definitions into the playbook (same as using the ansible-playbook '-e' option).

Example

Note that inside the ansible playbook, {{ version }}, {{ do_validation }}, {{ username }}, {{ password }}, and {{ host }} can be used to reference these extra variables that are passed in.
