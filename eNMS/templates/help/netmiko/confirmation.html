Netmiko Confirmation and Response

A regular expression that identifies an expected prompt from the device. Netmiko sends the corresponding response after the confirmation is matched in the device output.

Up to three confirmation/response pairs can be specified. Empty confirmations are ignored.

Example 1

Approve a File Copy Confirmation Request

Confirmation1:
Response1:

Notes:

  1. The device issues a confirmation prompt immediately after the file copy is initiated
  2. After sending the command, Netmiko Prompts Service waits until it sees text matching the Confirmation1 regex and then sends the Response1 text. If additional confirmation/response pairs are defined it will wait for a match against the the next confirmation and send the response text. Once all confiration/response pairs have been used, Netmiko Prompts Service waits for the device prompt before completing the service.
