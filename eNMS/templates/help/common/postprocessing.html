The postprocessing parameter provides a means for the user to modify the service results before the completion of the service. It also allows the user to inspect previously saved information with set_var and compare to information retrieved by this service to make further decisions as to what actions should be taken.

The postprocessing mode selector above allows the user to select when the postprocessing will run: always, only on success, or only on failure. Note that the postprocessing runs before the validation and notification parts of the service.
