The pre-processing parameter provides the ability to run python code to modify or check prior results.

Almost anything that might otherwise be done in post-processing from a prior service or services can also be done in the pre-processing. Note that the preprocessing can happen multiple times during retries in a workflow.

Example

Logging

For best visibility, we recommend that you use the 'log' function. You can also 'import logging'; however, those logs are only seen in the server logs log(severity, message, logger=logger) where severity is "info", "error", "warning", or "critical" and logger is omitted (default), "security", or some other user configured logger in settings
